Agis en tant que Lead Fullstack Developer. L'audit a révélé la cause de la page blanche : le payload JSON de l'inscription ne contient pas `statut_emploi`. Le frontend reçoit `undefined`, rate ses conditions, et redirige par erreur vers le `/dashboard`.

TÂCHES STRICTES DE CORRECTION :

1. BACKEND - MISE À JOUR DU MODÈLE (`User.php`) :
- Assure-toi que `statut_emploi` (et `gerant_id`) sont bien présents dans le tableau `$fillable` de ton modèle `User`.

2. BACKEND - RAFRAÎCHISSEMENT APRÈS CRÉATION (`AuthController::register`) :
- La méthode `User::create()` ne remonte pas automatiquement les valeurs par défaut de la base de données.
- Ajoute un `$user->refresh();` juste après le `create()` et avant le `return response()->json(...)`. 
- Cela garantira que le payload JSON contient bien `"statut_emploi": "aucun"`.

3. FRONTEND - PROGRAMMATION DÉFENSIVE (`App.jsx` / `Home`) :
- Même si le backend est corrigé, le routing frontend doit être tolérant aux erreurs.
- Modifie la condition de l'Ouvrier pour attraper le cas `undefined` :
```javascript
if (user.role === 'ouvrier') {
    // Si statut_emploi est 'aucun' OU s'il n'est pas encore défini dans le state
    if (!user.statut_emploi || user.statut_emploi === 'aucun') {
        return <PortailOuvrier />;
    }
    if (user.statut_emploi === 'en_attente') {
        return <WaitingApproval />;
    }
    if (user.statut_emploi === 'actif') {
        return <Navigate to="/dashboard" replace />;
    }
}
